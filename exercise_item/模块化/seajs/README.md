### 对象封装

### 划分私有空间

### 模块的维护扩展

- 模块的扩展一定要遵守一个约定：开闭原则
- 对添加开放、对修改封闭

### 模块的第三方依赖

- 不要直接在模块内部使用第三方依赖
- 模块与模块之间的依赖关系不够明显
- 最好通过 将 依赖项 注入 的形式 来解决第三方依赖的问题  依赖注入

### 总结

- 最大的问题，规范的问题
- 如果在多人协作开发过程中，会有很大的问题
- 多人协作开发过程中：代码的风格一定要统一

    + 解决了命名冲突
